==================== METHODS NOTES ====================

1. METHOD = REUSABLE BLOCK OF CODE THAT PERFORMS A TASK.
2. EACH QUESTION MUST BE A SEPARATE METHOD AND CALLED FROM main().
3. METHODS HAVE:
   - RETURN TYPE
   - NAME
   - PARAMETERS (INPUT)
   - BODY (LOGIC)
   - RETURN STATEMENT (IF NOT VOID)

-------------------- NUMBER METHODS (Q521–Q535) --------------------

4. DIGIT PROBLEMS USE LOOP WITH % 10 AND / 10.
5. REVERSE NUMBER BUILDS NEW NUMBER: rev = rev*10 + digit.
6. PALINDROME NUMBER COMPARES ORIGINAL WITH REVERSED.
7. PRIME CHECK USES DIVISIBILITY TEST UP TO sqrt(n).
8. FACTORIAL USES LOOP MULTIPLICATION; WATCH OVERFLOW FOR LARGE N.
9. NCR USES FACTORIALS: n! / (r!(n-r)!); VALIDATE r <= n.
10. GCD CAN BE EUCLID ALGORITHM; LCM = (a*b)/gcd.
11. POWER USING LOOP MULTIPLY a, b TIMES; HANDLE b=0 CASE.
12. ARMSTRONG CHECK: SUM OF EACH DIGIT^numberOfDigits.
13. PERFECT NUMBER: SUM OF PROPER DIVISORS = NUMBER.
14. STRONG NUMBER: SUM OF FACTORIAL OF EACH DIGIT.
15. NEON NUMBER: SUM OF DIGITS OF (n*n) EQUALS n.
16. AUTOMORPHIC: LAST DIGITS OF (n*n) MATCH n.

-------------------- OVERLOADING (Q536–Q537) --------------------

17. METHOD OVERLOADING = SAME NAME, DIFFERENT PARAMETERS.
18. OVERLOAD max() BY COUNT OR TYPE OF ARGUMENTS.
19. OVERLOAD area() BY DIFFERENT SHAPES (DIFFERENT PARAM LISTS).
20. COMPILER CHOOSES METHOD BASED ON ARGUMENT MATCH.

-------------------- STRING METHODS (Q538–Q541) --------------------

21. STRINGS ARE IMMUTABLE; BUILD RESULTS CAREFULLY.
22. COUNT VOWELS BY TRAVERSAL AND VOWEL CHECK.
23. REVERSE STRING USING LOOP OR StringBuilder.
24. PALINDROME STRING COMPARES FORWARD AND BACKWARD (IGNORE CASE).
25. ANAGRAM CHECK REQUIRES SAME LENGTH + SAME CHARACTER COUNTS.

-------------------- ARRAY METHODS (Q542–Q545) --------------------

26. ROTATION BY K REQUIRES K % n TO AVOID EXTRA ROTATIONS.
27. LEFT ROTATION MOVES FRONT ELEMENTS TO END.
28. RIGHT ROTATION MOVES LAST ELEMENTS TO FRONT.
29. SECOND LARGEST NEEDS TRACKING MAX AND SECOND MAX CAREFULLY.
30. REMOVE DUPLICATES CAN USE NESTED LOOPS OR TEMP ARRAY.

-------------------- MATRIX METHODS (Q546–Q550) --------------------

31. MATRIX = 2D ARRAY; ROW SUMS FIX ROW, LOOP COLUMNS.
32. COLUMN SUMS FIX COLUMN, LOOP ROWS.
33. UPPER TRIANGULAR: ALL ELEMENTS BELOW DIAGONAL MUST BE 0.
34. TRANSPOSE: swap a[i][j] with a[j][i] (NEW MATRIX OR IN-PLACE).
35. MULTIPLY MATRICES: RESULT[i][j] = SUM(A[i][k] * B[k][j]).

-------------------- BIT METHODS (Q551–Q554) --------------------

36. SET BITS COUNT: CHECK LAST BIT AND SHIFT/ DIVIDE.
37. BIT REVERSAL DEPENDS ON BIT LENGTH (8/16/32).
38. DECIMAL TO BINARY MANUAL: REPEATED DIVIDE BY 2 + STORE REMAINDERS.
39. BINARY TO DECIMAL MANUAL: PROCESS DIGITS USING PLACE VALUES (2^pos).

-------------------- SAFETY & EXCEPTION METHODS (Q555–Q556) --------------------

40. safeDivide(): CHECK DIVISOR == 0; RETURN -1 IF ERROR.
41. readIntSafe(): USE try-catch TO HANDLE INVALID INPUT.

-------------------- INTEGRATION / DEMO METHODS (Q557–Q570) --------------------

42. MENU-DRIVEN PROGRAM USES LOOP + SWITCH TO CALL METHODS.
43. PASS-BY-VALUE SWAP: SWAPPING INSIDE METHOD DOES NOT AFFECT ORIGINALS.
44. ARRAY MUTATION: CHANGES INSIDE METHOD AFFECT ORIGINAL ARRAY CONTENT.
45. STATIC COUNTER: STATIC VARIABLE RETAINS VALUE ACROSS CALLS.
46. RETURNING MIN/MAX USING ARRAY: RETURN ARRAY OF SIZE 2 OR OBJECT.
47. MULTIPLE MATRIX STATS: RETURN ARRAY/CLASS CONTAINING VALUES.
48. REUSABLE UTILITIES = GENERAL METHODS USED IN MANY PROGRAMS.
49. MEGA TOOLKIT = COMBINE ALL METHODS INTO ONE CLEAN MENU PROGRAM.
50. MAIN GOAL: CLEAN MODULAR CODE + REUSE + TESTING IN main().
